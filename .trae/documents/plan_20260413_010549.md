## 修复 MEGA 连接问题

根据错误日志和代码分析，问题出在前端与后端的会话密钥字段不匹配：

### 问题分析

1. **前端代码** (`useMegaStorage.ts`)：
   - 期望后端返回 `data.success` 和 `data.sessionId`
   - 代码：`if (data.success && data.sessionId) { return data.sessionId; }`

2. **后端代码** (`session/route.ts`)：
   - 实际返回 `data.sid` 而不是 `data.sessionId`
   - 代码：`return NextResponse.json({ success: true, sessionId: sessionId, ... });`

3. **错误结果**：
   - 前端无法获取到有效的会话密钥
   - 导致 `new Storage({ sid: sessionId })` 中的 `sessionId` 为 `null`
   - 最终抛出 `Not connected to MEGA` 错误

### 修复方案

1. **修改前端代码**：
   - 更新 `getMegaSession` 函数，同时支持 `sessionId` 和 `sid` 字段
   - 确保即使后端返回格式变化，前端也能正确处理

2. **增强错误处理**：
   - 添加更详细的错误信息
   - 确保连接失败时提供明确的用户反馈

3. **测试验证**：
   - 测试会话获取和 MEGA 连接
   - 验证文件上传功能

### 实施步骤

1. **修改 `useMegaStorage.ts` 文件**：
   - 更新 `getMegaSession` 函数，支持两种返回格式
   - 增强错误处理和日志

2. **测试修复效果**：
   - 测试会话获取 API
   - 测试 MEGA 连接
   - 测试文件上传功能

3. **验证修复**：
   - 确保上传过程中显示正确的进度
   - 确保文件成功上传到 MEGA
