# 存储中心用户说明书

## 系统概述

存储中心是一个功能强大的文件管理系统，支持本地存储和MEGA云存储集成，为用户提供安全、高效的文件管理体验。系统具有以下特点：

- 支持多种文件类型的上传、下载、删除和分享
- 智能存储策略：小文件使用本地存储，大文件自动使用MEGA云存储
- 实时上传进度监控和任务管理
- 响应式设计，支持桌面和移动设备

## 新功能介绍

### 1. MEGA云存储集成

系统现在集成了MEGA云存储服务，自动将超过5MB的大文件存储到MEGA云，解决了本地存储容量限制的问题。

**特点：**
- 自动识别文件大小，智能选择存储方式
- 大文件上传过程稳定可靠
- 支持大文件（最高1GB）的上传和下载

### 2. 流式上传技术

系统采用流式上传技术，避免了大文件上传时的内存溢出问题，提高了上传的稳定性和速度。

**技术优势：**
- 上传过程中内存占用低
- 支持断点续传
- 适用于网络条件不稳定的环境

### 3. 上传监控UI

新增了直观的上传监控界面，让用户实时了解上传状态。

**功能：**
- 实时显示上传进度和剩余时间
- 上传超时检测和错误提示
- 支持上传任务的重试和取消
- 上传任务历史记录

### 4. 任务管理系统

新增了上传任务管理功能，用户可以查看历史上传任务的详细信息。

**功能：**
- 查看任务状态（成功/失败/超时）
- 重试失败的上传任务
- 管理任务历史记录

## 操作流程

### 1. 文件上传流程

1. **选择文件**：点击页面上的"上传文件"按钮，选择要上传的文件
2. **智能存储分配**：系统自动根据文件大小选择存储方式
   - 小于5MB的文件：使用本地存储
   - 大于5MB的文件：使用MEGA云存储
3. **上传监控**：系统显示实时上传进度、剩余时间和状态
4. **任务完成**：上传完成后，文件会显示在文件列表中

### 2. 大文件上传操作

1. **准备文件**：选择大于5MB的文件
2. **开始上传**：系统自动将文件上传到MEGA云
3. **监控进度**：在上传过程中，可以查看实时进度和剩余时间
4. **查看结果**：上传完成后，文件会显示在文件列表中，标记为"MEGA"文件

### 3. 下载文件操作

1. **选择文件**：在文件列表中选择要下载的文件
2. **开始下载**：点击下载按钮
3. **监控进度**：系统显示下载进度
4. **完成下载**：文件自动保存到本地

### 4. 管理上传任务

1. **查看任务列表**：点击左下角的上传任务按钮
2. **查看任务详情**：在任务列表中查看任务状态和详细信息
3. **重试失败任务**：对失败或超时的任务，点击"重试"按钮重新上传
4. **关闭任务**：点击任务右侧的关闭按钮移除任务记录

## 常见问题和解决方案

### 1. 上传失败

**原因：**
- 网络连接不稳定
- MEGA服务器暂时不可用
- 文件大小超过限制

**解决方案：**
- 检查网络连接
- 点击"重试"按钮重新上传
- 确保文件大小不超过1GB

### 2. 上传超时

**原因：**
- 网络速度过慢
- 大文件上传时间过长

**解决方案：**
- 检查网络连接速度
- 尝试使用更快的网络
- 对特别大的文件，可以考虑分批次上传

### 3. 下载失败

**原因：**
- 网络连接不稳定
- 文件已被删除
- MEGA服务器暂时不可用

**解决方案：**
- 检查网络连接
- 确认文件是否存在
- 稍后重试

### 4. 存储容量不足

**原因：**
- 本地存储已满
- MEGA云存储已满

**解决方案：**
- 删除不需要的文件
- 清理回收站
- 考虑升级MEGA账户容量

## 性能优化说明

### 1. 内存使用优化

系统采用流式上传技术，避免了大文件上传时的内存溢出问题。上传过程中内存占用保持在低水平，即使上传1GB的大文件也不会导致系统崩溃。

### 2. 网络优化

- 采用分块上传技术，提高网络利用率
- 实现了上传超时检测，及时发现网络问题
- 支持断点续传，减少网络波动的影响

### 3. 用户体验优化

- 实时上传进度显示，让用户了解上传状态
- 预估剩余时间，帮助用户合理安排时间
- 清晰的错误提示，便于用户快速解决问题

## 系统要求

### 浏览器要求
- Chrome 90+、Firefox 88+、Safari 14+、Edge 90+

### 网络要求
- 稳定的网络连接
- 上传大文件时建议使用宽带网络

### 存储要求
- 本地存储：依赖浏览器localStorage，通常为5-10MB
- MEGA云存储：取决于用户的MEGA账户容量

## 部署说明

### 本地开发环境

1. 克隆代码仓库
2. 安装依赖：`npm install`
3. 配置环境变量：在`.env.local`文件中设置MEGA凭证
4. 启动开发服务器：`npm run dev`
5. 访问：`http://localhost:3000`

### 生产环境部署

1. 确保所有代码已提交到Git仓库
2. 配置Vercel项目，添加环境变量
3. 部署到Vercel
4. 访问部署后的URL

## 安全注意事项

- MEGA凭证存储在环境变量中，确保不会泄露
- 大文件上传过程中，建议在安全的网络环境下操作
- 定期清理不需要的文件，保持存储空间充足
