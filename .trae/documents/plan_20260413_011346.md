## 修复 MEGA 连接问题

### 问题分析

根据浏览器控制台的错误信息和代码分析，问题出在后端会话 API 返回的 `sessionId` 不是 MEGA 真正的 `sid`：

* **错误原因**：在 `app/api/mega/session/route.ts` 中，使用了 `storage.api.sid` 来获取会话密钥，但正确的属性应该是 `storage.sid`

* **后果**：`storage.api.sid` 返回 `undefined`，前端收到无效的会话标识，`megajs` 因此拒绝连接，报错 `starting a session without credentials isn't supported`

### 修复方案

1. **修改** **`app/api/mega/session/route.ts`**：

   * 将 `storage.api.sid` 改为 `storage.sid`

   * 同时返回 `sessionId` 和 `sid` 字段，确保前端兼容性

   * 延长过期时间到 24 小时

2. **验证修复效果**：

   * 测试会话 API 是否返回有效的 `sid`

   * 测试前端是否能成功连接 MEGA

   * 测试文件上传功能

### 实施步骤

1. **修改** **`app/api/mega/session/route.ts`**：

   * 修复 `sessionId` 获取方式

   * 增强错误处理

   * 优化返回格式

2. **测试修复效果**：

   * 访问 `http://localhost:3000/api/mega/session`，验证返回的 `sid`

   * 在前端尝试上传文件，验证连接和上传是否正常

### 预期结果

* ✅ 前端成功连接 MEGA

* ✅ 文件上传功能正常工作

* ✅ 上传进度正确显示

* ✅ 不再出现 `starting a session without credentials` 错误
