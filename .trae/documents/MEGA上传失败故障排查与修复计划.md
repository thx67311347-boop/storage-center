# MEGA上传功能修复计划

## 问题分析
测试脚本运行成功，说明MFA已关闭且MEGA账号可以正常登录。当前API路由存在以下问题：

1. **认证方式不一致**：测试脚本使用会话密钥，API路由使用邮箱密码
2. **文件处理方式**：API路由使用Buffer全量加载，可能导致内存溢出
3. **错误处理**：需要完善错误处理和日志记录

## 解决方案

### 1. 统一认证方式
- **使用会话密钥**：修改API路由使用MEGA_SESSION_ID进行认证
- **添加认证回退**：当会话密钥失效时，回退到邮箱密码认证

### 2. 优化文件处理
- **实现流式上传**：使用Readable.fromWeb()和.pipe()实现流式上传
- **避免内存溢出**：确保大文件上传时不会占用过多内存

### 3. 完善错误处理
- **添加详细日志**：记录完整的错误信息和上传过程
- **添加认证错误处理**：检测并处理认证失败的情况
- **添加网络错误处理**：处理网络超时和连接中断的情况

### 4. 测试验证
- **重启开发服务器**：确保代码更改生效
- **测试小文件上传**：验证基本功能
- **测试大文件上传**：验证流式上传稳定性
- **监控错误日志**：确保不再出现上传失败

## 实施步骤

1. **修改 `app/api/mega/upload/route.ts`**：
   - 使用会话密钥认证
   - 实现流式上传
   - 完善错误处理

2. **重启开发服务器**：`npm run dev`

3. **测试上传功能**：
   - 上传小文件（<1MB）
   - 上传中等文件（10-50MB）
   - 上传大文件（>100MB）

4. **验证修复结果**：
   - 检查服务器日志
   - 确认文件成功上传到MEGA
   - 验证前端错误提示是否正确

## 技术要点
- 使用会话密钥绕过MFA认证
- 实现流式上传避免内存溢出
- 完善错误处理和日志记录
- 确保前后端数据结构一致

## 预期成果
- MEGA上传功能恢复正常
- 稳定的大文件上传能力
- 详细的错误日志便于排查
- 前端错误提示清晰准确
