# 修复文件上传和下载功能的问题

## 问题分析

### 上传问题
- **原因**：当前系统使用本地 blob URL (`URL.createObjectURL(file)`) 作为文件存储方式
- **表现**：文件看似上传成功并显示在界面上，但实际只是临时的本地URL
- **影响**：当页面刷新或组件卸载时，blob URL会被清理，导致文件无法访问

### 下载问题
- **原因**：下载逻辑尝试从临时的blob URL获取文件
- **表现**：系统显示下载完成，但在计算机中找不到文件
- **影响**：用户无法实际获取上传的文件

### 缺少下载进度显示
- **原因**：当前下载逻辑没有实现进度监控
- **表现**：用户无法看到下载进度，体验不佳

## 修复方案

### 1. 改进文件存储方式
- **方案**：使用浏览器的 IndexedDB 或 localStorage 存储文件数据
- **实现**：
  - 创建文件存储服务，将文件数据转换为 base64 或 ArrayBuffer 存储
  - 为每个文件生成唯一标识符
  - 保存文件元数据和数据到本地存储

### 2. 修复下载功能
- **方案**：从本地存储中读取文件数据并创建可下载的blob
- **实现**：
  - 从存储中获取文件数据
  - 将数据转换回 blob 对象
  - 创建临时下载链接并触发下载

### 3. 添加下载进度显示
- **方案**：实现文件下载的进度监控
- **实现**：
  - 创建下载进度状态管理
  - 使用 `XMLHttpRequest` 或 `fetch` API 监控下载进度
  - 显示实时下载百分比和进度条

## 修复步骤

1. **创建文件存储服务**：
   - 创建 `useFileStorage.ts` 中的文件存储和读取函数
   - 实现文件数据的持久化存储

2. **修改文件上传逻辑**：
   - 更新 `useFileOperations.ts` 中的 `handleFilesUploaded` 函数
   - 使用新的存储服务保存文件数据

3. **修复文件下载逻辑**：
   - 更新 `useFileOperations.ts` 中的 `handleFileDownload` 函数
   - 从存储中读取文件数据并创建可下载的blob

4. **添加下载进度显示**：
   - 创建下载进度组件
   - 实现进度监控和显示

5. **测试验证**：
   - 上传文件并验证文件显示在界面上
   - 刷新页面后验证文件仍然存在
   - 下载文件并验证文件能正确保存到计算机
   - 验证下载进度显示功能

## 技术实现要点

- 使用 `FileReader` API 将文件转换为 base64 或 ArrayBuffer
- 使用 `localStorage` 或 IndexedDB 存储文件数据
- 使用 `fetch` API 或 `XMLHttpRequest` 监控下载进度
- 实现文件元数据和数据的关联存储
- 确保文件存储和读取的性能优化
