## Fix MobileLayout Import Issue in Settings Page

### Issue
The `MobileLayout` component is not being imported correctly in `app/settings/page.tsx`, causing TypeScript errors.

### Solution
1. **Update the import statement** in `app/settings/page.tsx` to use the absolute path alias defined in tsconfig.json:
   - Change from: `import MobileLayout from '../components/layout/MobileLayout';`
   - Change to: `import MobileLayout from '@/app/components/layout/MobileLayout';`

2. **Verify the fix** by checking for TypeScript errors

### Files to modify
- `app/settings/page.tsx`

### Expected result
- TypeScript error resolved
- Settings page renders correctly
- MobileLayout component is properly imported and used
